Artifact: diagrama de dominio
Un diagrama de dominio en software es una representación visual que muestra las entidades clave y las relaciones entre ellas dentro de un determinado dominio o contexto. El diagrama de dominio se utiliza para comprender la estructura y la organización de un sistema, identificar las entidades relevantes y cómo se relacionan entre sí.

El objetivo principal de un diagrama de dominio es modelar el dominio del problema que el software está abordando. Ayuda a los desarrolladores y a las partes interesadas a tener una visión clara de los conceptos clave, las interacciones y las reglas del negocio que influyen en el diseño y desarrollo del software.
